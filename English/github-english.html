GitHub英语 · 单词+场景语句

核心基础词汇

Create a new repository for the project.
为项目创建一个新仓库。
repository (repo) 仓库
场景：初始化项目第一步
Clone this repo to your local machine.
将这个仓库克隆到本地机器。
Clone 克隆
指令：git clone [仓库地址]
Fork the repository to your own GitHub account.
将仓库复刻到自己的GitHub账号。
Fork 复刻/派生
场景：贡献开源项目第一步
Star this repo if you find it useful.
如果你觉得有用，给这个仓库点星标。
Star 收藏/星标
Watch the repo to get notifications of updates.
关注仓库以接收更新通知。
Watch 关注/监控
Create a new branch for the new feature.
为新功能创建一个新分支。
Branch 分支
指令：git checkout -b [分支名]
Make a commit with a clear message.
提交代码并填写清晰的提交信息。
Commit 提交（代码）
指令：git commit -m "feat: add login function"
Push your local changes to the remote repo.
将本地修改推送到远程仓库。
Push 推送
指令：git push origin [分支名]
Pull the latest code from the main branch.
从主分支拉取最新代码。
Pull 拉取
指令：git pull origin main

协作开发

Open an issue to report a bug.
新建Issue反馈一个bug。
Issue 问题/工单
Create a pull request (PR) to merge your branch into main.
创建PR将你的分支合并到主分支。
Pull Request (PR) 合并请求
Review the code changes in the PR.
审核PR中的代码修改。
Review 审核（代码）
Merge the feature branch into the main branch.
将功能分支合并到主分支。
Merge 合并（分支）
场景：PR审核通过后执行
Resolve the merge conflict before merging.
合并前解决代码冲突。
Conflict 冲突（代码）
Leave a comment on the PR to ask questions.
在PR下留言提问。
Comment 评论/备注
Approve the PR if the code is correct.
代码无误则批准这个PR。
Approve 批准
Address the comments in the PR.
处理PR中的审核意见。
Address 处理/解决

代码管理

Write a meaningful commit message.
编写有意义的提交信息。
Commit message 提交信息
规范：feat/fix/docs/style/refactor/test/chore
Revert the last commit to undo the changes.
回滚上一次提交以撤销修改。
Revert 回滚（提交）
指令：git revert [commit-hash]
Create a tag for the v1.0 release.
为v1.0版本创建标签。
Tag 标签（版本）
指令：git tag -a v1.0 -m "Release v1.0"
Create a new release from the v1.2 tag.
基于v1.2标签创建新版本发布。
Release 发布（版本）
Fetch the repo before pulling to avoid conflicts.
拉取前先抓取仓库以避免冲突。
Fetch 抓取（远程更新）
指令：git fetch origin
Checkout the main branch to test the code.
切换到主分支测试代码。
Checkout 切换（分支）
指令：git checkout main
Delete the feature branch after merging.
合并后删除功能分支。
Delete 删除（分支）
指令：git branch -d [分支名]（本地）| git push origin --delete [分支名]（远程）
Sync your fork with the original repository.
将你的复刻仓库与原仓库同步。
Sync fork 同步复刻仓库
